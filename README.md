# 大宋模拟器 · Da Song Simulator

一个以真实历史为底本的**大宋皇帝沙盘推演**游戏。你扮演北宋皇帝，从**熙宁元年（1068 年）**开局，
每回合推进真实世界**一个月**：颁布经济、科技、军事、政治各方面的政策，由接入的 **LLM 推演引擎**
（支持 OpenAI 与 Anthropic 两种格式）根据历史规律与当前世界状态进行严肃推演，结果写入数据库，
下一回合在「既有状态 + 新政策」之上继续演化。影响范围是**全球** —— 地图上可见大宋及周边
十六个势力的领土、经济、军事、科技、政治状况，以及它们对大宋的外交态度。

> 推演会权衡**政策实施难度**（财政、吏治、既得利益、民意、地理），并允许出现**意料之外的结果**
> （执行走样、天灾、边衅、党争、外敌反应等），不会一味顺你心意。

## 截图说明
- **国情**：当前所选势力的经济/科技/军力/政治稳定/民生/兵力仪表盘 + 国势走势曲线。
- **列国**：各势力横向对比表，含对宋外交关系。
- **政策**：勾选系统建议 + 自由输入诏令 → 「推演本月」。
- **史册**：历代纪事编年。
- **地图**：真实世界底图叠加各势力领土（点击/悬停查看详情）。
- **存档 / 读档**：💾 命名存档/读取/删除，保存整局（所有回合状态 + 史册），重开不清除存档。
- **回溯（时间旅行）**：「史册」里每个历史回合都有「↩ 回到此回合」（含「回到开局」），可回到任意过往回合从该处重新演化——会丢弃其后回合并正确还原国策账本（撤销其后颁布/废止的国策）。
- **文言 / 白话**：推演叙事同时生成文言与白话两版，弹窗与史册右上角一键切换（默认白话，易读）。
- **演化视频**：🎬 把每回合的世界快照（地图按国力着色 + 日期 + 国力榜 + 当月纪事）渲染成一段 mp4，方便做"世界随时间演化"的视频（后端 matplotlib + ffmpeg；另有 `GET /api/timeline` 导出逐回合原始数据 JSON）。

## 全球势力（23 方）
- **东亚/内亚**：北宋(玩家)、辽、西夏、吐蕃诸部、大理、高丽、大越(李朝)、日本、喀喇汗。
- **西亚/欧洲**：塞尔柱、拜占庭、法蒂玛、神圣罗马、伽色尼、基辅罗斯、注辇(朱罗)。
- **美洲(印第安)**：托尔特克(中美洲)、密西西比诸部(北美·卡霍基亚)、奇穆(南美安第斯)。
- **非洲撒哈拉以南**：加纳帝国(西非黄金之国)、阿比西尼亚(东非)、马蓬古布韦(南部非洲)。
- **大洋洲**：澳洲原住民。

> 领土由真实国界几何裁剪而成；中国本土按经纬度切分为宋/辽/西夏/吐蕃/大理，
> 内部边界用**分形中点位移**生成「犬牙交错」的自然边界（非直线方框）。

## 快速开始

```bash
./run.sh              # 首次会自动建虚拟环境、装依赖，然后启动
# 浏览器打开 http://127.0.0.1:8000
```

默认使用**离线推演引擎**（`mock`，无需任何 API Key 即可完整试玩）。

### 接入真实 LLM

点界面右上「⚙ 设置」，或编辑 `config.local.json`（参考 `config.local.json.example`）：

| provider | 说明 | model 示例 | base_url |
|---|---|---|---|
| `mock` | 离线启发式推演，无需密钥 | — | — |
| `anthropic` | Anthropic Messages API | `claude-opus-4-7` | 留空=官方 |
| `openai` | OpenAI 兼容接口（OpenAI / DeepSeek / 本地 vLLM / Ollama 等） | `gpt-4o` / `deepseek-chat` | 如 `https://api.deepseek.com/v1` |

也可用环境变量覆盖：`LLM_PROVIDER` `LLM_MODEL` `LLM_API_KEY` `LLM_BASE_URL` `LLM_TEMPERATURE` `LLM_MAX_TOKENS`。

> Anthropic 路径已对稳定的系统提示启用 **prompt caching**，多回合可降低输入成本。

### 现行国策：影响系数 + 时间衰减（持续生效）
政策是长期国策、非一次性事件。为避免把所有历史政策原文逐回合塞进提示词（拖慢且加重 LLM 负担），采用**量化系数模型**：
- 颁布国策时，LLM（或离线 mock）为其量化一组**月度影响系数 `coeffs`**（各项国情的增量，可正可负）与**衰减因子 `decay`(0~1)**。
- 每项系数可为**标量**（颁布初期较强、随后渐衰至 0），或 **`{init, steady}` 时间剖面**（由初期值平滑过渡到长期稳定值；**steady 永久保持、绝不衰减**，可变号）——用于「先投入后回报」（研发/兴学：初期烧钱→后期回本）与「先小后大、持续收益」（口岸/市舶：收益逐月走高并长期维持）。
- 政策的"快慢"由 **`half_life`（成熟半衰期·月）** 控制：赈济等速效用 2~5；常规新政 8~18；制度/基建/文教/科研等**大后期**政策用 30~80——前期几乎不显效、要数年乃至十数年才逐步释放威力，steady 长期不衰。例：广兴学校 half_life=36，科技效力 7 年间由 0.6 缓升向 2.5，国库约 7 年后才由亏转盈。
- 此后每回合，引擎用公式 `effect(m)=steady+(init-steady)·decay^(m-1)`（`decay=0.5^(1/half_life)`，标量则 `steady=0`）**确定性地自动累计**，**无需再调用 LLM、也不再把旧政文本喂给模型**。
- LLM 返回的 JSON 若有未转义引号等常见错误，解析失败时用 `json-repair` 自动修复兜底（并在提示词中要求改用中文引号）。
- 提示词里只保留各国策的**标题 + 推行月数**（极简，供叙事参考）；新政的即时冲击仍由 LLM 给 `changes`。
- 国策可被 LLM 标记为废止/完成（`ended_policies`）而退出账本。
- 界面「政策」页的「现行国策」栏实时显示每条国策当月的（已衰减）效力。

### 回合数据流
1. 玩家提交政策文本 + 勾选项 → `POST /api/turn`
2. 引擎读取当前回合各势力状态 + 近期纪事
3. 组装系统提示 + 世界状态 → LLM 返回**严格 JSON**（叙事 / 事件 / 各势力数值增量 / 下回合可选项 / 难度评议）
4. 引擎对增量做范围钳制后写入「下一回合」快照，推进月份，记录史册
5. 前端刷新仪表盘、地图、史册，并弹出推演纪事

### 主要 API
`GET /api/state`·`POST /api/turn`·`GET /api/history`·`GET /api/series/{key}`·
`POST /api/reset`·`GET|POST /api/config`·`GET /api/{world,factions}.geojson`·

## 时间单位：一回合 = 一年
每个回合推演真实世界**一整年**。所有随时间的流量都按年计：科技点基础 ≈ **18 点/年**（按 1068→今约 958 年 / 全树 16790 点校准）、国库按年税收结算、政策系数与半衰期(half_life)单位均为「年」。

## 地图随推演动态扩张
LLM 在推演中**自主判断**是否发生疆域变更：当大宋（或他国）实控一片**无主之地**（如台湾、流求、西域屯田）或**吞并败国**时，会在 `territory_changes` 给出 owner+名称+经纬度 bbox，引擎与陆地求交后**重绘地图**（如占台湾后台湾染为宋色）。

## 经济·军事·科技·战争（确定性模拟层）
在 LLM 定性推演之外，引擎每回合对**所有势力**做确定性结算（`game/sim.py`）：
- **国库收支**：`税收(经济×12) − 军费(兵力×武器系数×5) − 行政(人口×0.05)`，让国库消耗有据可依（国情页透明展示）。
- **军力量化**（`game/sim.py`）：`军力 = 兵力 × 武器系数(随科技) × 民族战力`；民族战力由开局数据反推（如辽骑兵战力高），军力随兵力/科技联动；军费消耗国库。
- **破产**：国库 ≤ 0 → 军队欠饷哗变(兵力流失)、民生凋敝、政局动荡；敌我一视同仁。
- **列国**页显示每个势力的国库、军力、对宋关系，并标注「破产/沦陷」。
- **科技树**（`game/tech.py`，参考文明6精简）：自宋代工巧（活字印刷/火药/指南/占城稻）起步，历革新→工业→现代→信息·智能，终于 **大模型 AI**（文明6所无）。按国力积累科技点，选向研究，完成解锁并把加成永久落到国情。
  - **科研速率按真实时间轴校准**：1068→今约 958 年(11496 月)、全树约 16790 点，故基础速率 ≈ **1.5 点/月**（开局研一项约十数年，符合史实），随经济/科技增长而加快。
  - **政策可影响科研速度**：兴学、设格物院、开科取士、奖掖工巧等政策会带 `science` 系数（每月额外科技点，多少由 LLM 裁决，可正可负）；科技页显示「基础 + 国策」拆分与"预计 N 年解锁"。
- **战役（纯公式结算，`game/war.py`）**：「列国」可对某国**开战**，设投入兵力与战役强度（袭扰/会战/决战）。引擎按
  `战力 = 投入兵力 × 武器系数(科技) × 民族战力 × 士气` + 守方守土优势(×1.25) + 战场偶然，算出胜率并据此折算双方伤亡、军费、兵燹之损、外交恶化；守军军力归零即被占领。开战弹窗有"投入战力 vs 守军"实时估算。
  （北宋兵多质弱，须以优势兵力进攻；这也还原了宋伐夏屡挫的史实。）
- **战与和合一**：「列国」每国只有一个按钮——**和平时显示「开战」，交战中显示「议和」**（行内标「交战」）。
- **议和**：按面积**割地**(直接重绘地图)+**赔款**(改两国国库)，可选大宋战胜(对方割地赔款)或战败(大宋割地赔款)，议和即息兵。

## 数值维度
国库(万贯)、人口、经济指数、军力指数、兵力、科技(0-100)、政治稳定(0-100)、民生(0-100)、对宋外交(-100~100)。

## 已知简化 / 可扩展方向
- 领土多边形为**近似高亮**，非精确历史疆界；领土易手目前体现在数值与叙事，尚未动态改色（接口已预留）。
- 离线 `mock` 引擎仅基于关键词启发，真实开放式推演请接入 LLM。
- 可继续扩展：战争结算、领土动态变更上色、随机历史事件库、多皇帝存档、科技树、贸易网络等。
